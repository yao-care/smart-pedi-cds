# 資安事件應變聯絡清單

> ISO/IEC 27001:2022 控制點 **A.5.26 — 資安事件之回應**（聯絡窗口定義）
> 對象系統：Smart Pedi 兒童發展評估
> 文件版本：1.0　建立日期：2026-06-10　覆核週期：每 6 個月

營運主體：**藥提醒科技有限公司**

## 1. 內部應變窗口

本系統為**單人維運**，事件指揮、技術修補、資料保護／合規三職由同一人兼任。

| 角色 | 職責 | 職稱 | 聯絡方式 | 備援 |
|---|---|---|---|---|
| **主要聯絡人 / 事件指揮**（兼技術修補、資料保護／合規） | 接收通報、判定嚴重度、決策圍堵與還原、根因分析與修補、評估個資衝擊與合規通報 | 系統維運負責人（藥提醒科技有限公司） | lightman.chang@gmail.com / 0988-911679 | 單人維運，無內部備援；重大事件委由外部資安協處窗口（見 §2） |

> ⚠️ 單人維運為**單點失聯風險**。建議至少指定一名緊急備援聯絡人（即使非技術人員，亦可在主要聯絡人失聯時對外啟動 §2 外部窗口）。

## 2. 外部窗口

| 對象 | 用途 | 聯絡方式 |
|---|---|---|
| **GitHub Support** | Pages 部署異常、帳號遭盜用、惡意 commit 處置 | https://support.github.com/ |
| **網域登記人 / 註冊服務** | DNS／網域挾持還原 | 藥提醒科技有限公司（`yao.care` 登記人；註冊服務商帳號由該公司持有，經維運負責人操作） |
| **FHIR 主機方（醫院 standalone）** | token 洩漏時撤銷 client、旋轉密鑰 | 尚未對接（目前無醫院 standalone 上線；上線時補登） |
| **GCM 收案平台方** | GCM PKCE 動態註冊相關事件 | 已建立對接窗口；事件聯繫經維運負責人轉介（如需列對方直接窗口 email／電話可後補） |
| **主管機關通報窗口** | 個資外洩法定通報（如適用） | 依《個人資料保護法》向目的事業主管機關通報，由藥提醒科技有限公司判定具體窗口 |
| **資安事件協處** | 通報與技術協處 | TWCERT/CC：https://www.twcert.org.tw |

## 3. 通報啟動條件

- **P1/P2 事件**（見 [事件應變計畫](incident-response-plan.md) §2）：立即電話 + email 通知主要聯絡人，1 小時內未回應則啟動備援聯絡人。
- 涉及病患個資疑似外洩：同步通知資料保護／合規窗口，評估是否觸發法定通報時限。

> 內外部窗口已填實。**建議補強**：(1) 指定一名緊急備援聯絡人以降低單點失聯風險；(2) 如需，補登 GCM 收案平台對方直接窗口與目的事業主管機關通報窗口。
