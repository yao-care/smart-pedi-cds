# P3: AI Alert Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Implement the complete client-side AI alert engine — rule engine, baseline calculation, ML inference — all running in Web Workers to avoid blocking the UI.

**Architecture:** Three Web Workers (rule-engine, baseline, ml-inference) communicate with main thread via postMessage. Workers use Transferable Objects for performance.

**Tech Stack:** Web Workers, js-yaml, ONNX Runtime Web (WASM), TypeScript

---

### Task P3-1: Rule Engine Worker

**Files:**
- Create: `src/engine/workers/rule-engine.worker.ts`

**Requirements:**
- Receives: patient observations, rule YAML (parsed), age group
- Evaluates 7 indicators against 3-level thresholds (normal/advisory/warning)
- Critical = exceeds warning range (implicit)
- Multi-indicator escalation: N+ advisories → warning (configurable)
- Trend detection: consecutive days of worsening → escalation
- Deduplication: same patient+indicator within window → skip
- Output: per-indicator risk levels + rationale strings
- Use `self.postMessage()` / `self.onmessage` pattern

---

### Task P3-2: Baseline Calculation Worker

**Files:**
- Create: `src/engine/workers/baseline.worker.ts`

**Requirements:**
- Receives: observation history for a patient
- Calculates personal mean, std per indicator
- Minimum 20 samples required, else return null (fallback to population baseline)
- Rolling update: incorporate new observations incrementally
- Z-score calculation: (value - mean) / std, handle std=0 → z-score=0
- Output: baseline parameters + z-scores for latest observations

---

### Task P3-3: ML Inference Worker

**Files:**
- Create: `src/engine/workers/ml-inference.worker.ts`

**Requirements:**
- Load ONNX model from `/smart-pedi-cds/models/risk-model.onnx` (lazy, first use)
- ONNX Runtime Web with WASM backend
- Input: 7-element z-score vector (Float32Array)
- Output: predicted risk level + probability distribution
- Cache model session after first load
- Handle missing z-scores (skip those features or use 0)
- Use Transferable Objects for Float32Array

---

### Task P3-4: Engine Orchestrator

**Files:**
- Create: `src/engine/risk-analyzer.ts`

**Requirements:**
- Coordinates all 3 workers
- Flow: observations → baseline worker → z-scores → rule engine + ML worker → merge results
- Merge strategy: take higher severity (rules vs ML)
- Generate complete RiskAnalysisResult with explainability data
- Interface:
  ```typescript
  interface RiskAnalysisResult {
    patientId: string;
    overallRisk: RiskLevel;
    ruleResult: { level: RiskLevel; rationale: string; indicators: IndicatorResult[] };
    mlResult: { level: RiskLevel; probabilities: Record<RiskLevel, number> };
    baselines: Record<string, { mean: number; std: number; zScore: number }>;
    timestamp: Date;
  }
  ```
